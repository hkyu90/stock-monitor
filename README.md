# 📊 Stock Monitor

주식 매수추천 & 관심종목 모니터링 — 데이터 기반 스코어링 + 매수/매도 타이밍 시그널

|---|---|---|

## 사용법

```bash
# 의존성 설치
pip install -r requirements.txt

# 일일 스크리닝 리포트 (옵시디언 자동 저장)
python main.py daily

# 보유종목 매도 시그널 + 관심종목 매수 시그널
python main.py watchlist

# 단일 종목 스코어링
python main.py score NVDA NASDAQ
python main.py score 005930 KOSPI

# 백테스팅
python main.py backtest NVDA NASDAQ --strategy midterm
python main.py backtest 005930 KOSPI --strategy longterm
```

## 심층 분석 체계

### 거시경제 환경 분석
| 지표 | 소스 | 활용 |
|------|------|------|
| S&P500 / KOSPI / NASDAQ 추세 | yfinance | 시장 전체 방향성 |
| VIX (공포지수) | yfinance | 시장 변동성 / 리스크 수준 |
| USD/KRW 환율 | yfinance | 원화 강약세 → 해외 투자 영향 |
| 미국 10년물 국채 금리 | yfinance | 금리 환경 → 주식 밸류에이션 영향 |

### 종목별 보유/매도 판단
- **추세 강도 (ADX)**: 상승/하락 추세의 강도와 방향, 전환 신호 감지
- **지지/저항선**: 피봇 포인트 + 이동평균선 기반 핵심 가격대
- **펀더멘털 방향성**: ROE/매출성장률/EPS성장률/영업이익률 종합 → 개선/유지/악화
- **섹터 상대강도**: 해당 시장 대비 상대적 강약

### 판단 등급 (약세장 축적 철학 반영)
| 판단 | 의미 | 액션 |
|------|------|------|
| 🟢➕ 추가 매수 | 펀더멘털 강건 + 조정/지지선 근접 | 지지선 분할 매수, 12개월+ 보유 |
| 🟢 적극 보유 | 추세+펀더멘털+매크로 모두 우호 | 중장기 보유 (6개월+) |
| 🔵 보유 유지 | 대체로 양호, 일부 리스크 | 2~4주 관찰, 추세 확인 |
| 🟡 관망 | 혼재된 신호 | 1~2주 내 추세 확인 |
| 🟠 부분 매도 | 리스크 요인 우세 | 보유 물량 30~50% 정리 |
| 🔴 전량 매도 | 펀더멘털 악화 + 추세 붕괴 | 가능한 빨리 정리 |

**가중치 원칙**:
- **펀더멘털 방향이 최우선** (±4) — ROE/매출성장/EPS/영업이익률
- 추세는 ADX 30+ 에서만 강한 페널티 (±3)
- 매크로는 보조 (±1), **펀더멘털 강건 종목은 매크로 페널티 면제**
- **폭발적 성장 프리미엄** — 매출 +40% 이상 시 고성장 가중치 추가
- 지지선 이탈 시 강건 종목은 "축적 구간"으로 재해석

## 스코어링 체계

| 점수 | 시그널 | 액션 |
|------|--------|------|
| 80~100 | 강력매수 | 즉시 매수 검토 |
| 65~79 | 매수관심 | 관심종목 등록, 진입점 대기 |
| 50~64 | 중립 | 보유 유지 |
| 35~49 | 매도관심 | 손절/익절 검토 |
| 0~34 | 강력매도 | 즉시 매도 검토 |

## 리포트 출력

- 옵시디언 볼트: `06_개인/주식모니터링/`
- 매일 오전 8시(KST) Windows Task Scheduler 자동 실행
- 보유종목별 수익률 + 매도 시그널 (익절/손절/기술적 시그널)
- 포트폴리오 요약 (총 투자원금, 평가액, 수익률)

```
│   │   └── momentum.py       # 모멘텀 (52주, 수급, 상대강도)

│   ├── backtest/backtester.py # 백테스팅 엔진
│   └── report/generator.py   # 옵시디언 리포트 생성
├── main.py                   # CLI 진입점
├── run_daily.bat             # 자동 실행 배치
├── setup_schedule.bat        # Task Scheduler 등록
└── requirements.txt
```
